웹사이트 연습작 설계


요구사항 

<라이브 스트리밍 기능>
- 무작위 인원의 유저들이 실시간 동영상 시청 가능
- Adaptive Streaming 사용. (HLS)
- 지원 가능한 영상 품질 및 해상도는 임대한 서버 한계치에 맞춰서.   
- 1명의 스트리머와 무작위 인원의 시청자로된 구성 까지만 구현.


<유저>
- 회원가입시 인적사항은 이름, 주소, 전화번호, 이메일로 간략하게 구현.
- 유저 Password 기본적 보안
- 소셜 로그인 1개까지 지원 가능
- 게스트 로그인 가능.
- 모든 유저는 0원 이상의 인터넷 화폐를 보유한다.
- 현금/카드 화폐를 구매하는 기능은 일단 생략


<유저 활동>
1. 단순 상호작용
- 다른유저 에게 화폐 보내기 기능
- 귓속말 기능
- 대화 및 시청 차단기능

2. 라이브 스트림 중계/시청
- 중계/시청 합해서 한번에 하나의 활동만 가능.
- 모든 시청자는 실시간 채팅 참여 가능 
- 스트리머는 시청자 무작위 강퇴 가능. 영구 블럭과 24시간 밴까지 구현.
- 회원은 좋아요/싫어요 두가지의 스트림 평가 가능.

3. 영상 업로드
- MP4 포맷의 영상만 업로드 가능.
- 테스트상 유저당 총 영상 50Mb 까지만 제한.

4. 게시글 업로드
- 기본적인 텍스트 에디터 플러그인으로 구현
- 좋아요/싫어요 및 댓글 가능


개체 분석

서버 
- 비디오 스트림들 (갯수 미정)
- 간단한 정보통신용  HTTP 요청/응답
- 채팅용 소켓통신모듈

클라이언트
- 서버로 HTTP 전송/요청하는 UI/스크립트.
- 서버에서 HTTP 전송받아 출력하는 UI
- 채팅용 소켓통신
- 실시간 스트림 가능 비디오 플레이어

개체들
 
1. 유저
	- 개인정보 (Primary: 이메일)
	- 화폐보유량
	- 게시글
	- 업로드한 영상들
	- 응답한 게시글 ID
	- 팔로우 정보

2. 스트림
	- 방송 고유번호
	- 방송인
	- 시청자 목록
	- 밴 리스트
	- 영상 스트림 전송 정보
	- 채팅용 소켓 정보
	- 채팅 로그
3. 영상
	- 기본정보 (중계시작시간,업로드시간)
	- 누적 시청자수
	- 댓글
	- 대댓글

4. 게시글 
	- 게시글 ID
	- 게시글 내용,정보 
	- 댓글및 관련정보 (대댓글은 Depth및 Parent ID로 구별)

관계들 

시청 (유저:스트림,영상,게시글; 1:1)

송금 (유저:유저; 1:1)

메세징 (유저:유저: 1:N)




화면 구성

1. 공통
	- 네비게이션 바 (검색폼, 로그인 버튼, 유저용 드랍다운)
2. 메인 화면
	- 팔로잉 중인 채널의 최신 스트림/영상 
 	- 실시간 중계 영상 목록 TOP 5
	- 지난 하루동안 가장 많이 시청한 영상 TOP
	- 게시글 조회수 top5

3. 검색 결과 화면 
	- 관련 유저/스트림/게시글/영상 목록
	- 라이브 중인지 표기할 것





4. 유저 개인 페이지
	- 사이드 내비게이션바 (팔로잉 중인 채널, 개인정보, 활동내역 선택가능)
	- 컨텐츠 섹션 (팔로잉중션인 채널 정보 / 개인정보 / 활동내역중 하나보여줌) 
5. 라이브 스트리밍 페이지
	- 영상 섹션
	- 채팅 섹션
	
6. 영상 페이지
	- 영상 섹션
	- 댓글 섹션

7. 게시글 페이지 
	- 게시글 섹션
	- 댓글 섹션
